# TypeC 0.1.0 Release Candidate

Release-candidate tag: `typec-0.1.0-rc.1`

This document records the TypeC 0.1.0 release-candidate freeze. It is valid only for a compiler
state that passes the full Phase 300 validation gate.

## Frozen diagnostic inventory

The TypeC 0.1 diagnostic code names and numeric code strings are frozen in
`src/core/diagnostic_codes.ts` and documented in `docs/diagnostics.md`. Tooling may rely on the code
names and `E####` strings listed for 0.1.

## Frozen C ABI surface

The TypeC 0.1 emitted C ABI for supported runtime layout types is frozen in `docs/c-emission.md`.
Primitive TypeC names map one-to-one to emitted C typedef names. The scalar prelude is part of the
0.1 ABI contract.

## Required validation gate

```bash
deno fmt --check
deno test -A
deno task build
./bin/STC --version
./bin/STC check examples/0.1/main.tc
./bin/STC build examples/0.1/main.tc
./bin/STC run examples/0.1/main.tc
```

Expected version output:

```txt
TypeC 0.1.0
```
