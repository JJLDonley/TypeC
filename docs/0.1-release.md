# TypeC 0.1 Release Notes

Release-candidate freeze record: `docs/0.1-release-candidate.md`.

TypeC 0.1.0 is the first self-contained TypeC language release target. It defines a strict,
statically checked systems-language subset with TypeScript-like syntax and deterministic C11
emission.

TypeC 0.1 is not JavaScript and is not TypeScript with a runtime. It has no garbage collector,
prototype chain, dynamic object model, implicit `undefined`, truthiness, `any`, promises, or module
loader.

## Install and build

Run from the repository root with Deno installed:

```bash
deno task check
deno task lint
deno test -A
deno task build
./bin/STC --version
```

Expected version output:

```txt
TypeC 0.1.0
```

## Compiler commands

```bash
deno run -A src/driver/main.ts check examples/0.1/main.tc
deno run -A src/driver/main.ts build examples/0.1/main.tc
deno run -A src/driver/main.ts emit-c examples/0.1/main.tc
```

After `deno task build`, the compiled CLI is available as `./bin/STC`.

## Supported 0.1 features

The authoritative language reference is `docs/language.md`. The supported 0.1 surface includes:

- `.tc` source files, line comments, block comments, numeric literals, string literals, and static
  template literals
- primitive runtime layout types: `bool`, `b8`, `i8`, `i16`, `i32`, `i64`, `u8`, `u16`, `u32`,
  `u64`, `usize`, `f32`, `f64`, and `void`
- `const`, `let`, return, blocks, `if`, `else`, `while`, `do while`, `for`, `for..of`, `for..in`,
  `switch`, `break`, `continue`, and `defer`
- arithmetic, comparison, logical, bitwise, conditional, nullish coalescing, optional chaining,
  non-null assertion, casts, calls, field access, indexing, and static assignments
- functions, extern functions, rest parameters, optional parameters, default parameters, overload
  declarations, and function-typed values in supported value positions
- records, structs, record literals, field assignment, object spread/rest, arrays, tuples, slices,
  and aggregate literals
- enums, tagged unions, optional values, and literal-only static aliases where documented
- classes with static value layout, constructors, fields, methods, single inheritance flattening,
  method override replacement, and nominal `implements` checks
- interfaces as static contracts and explicit borrowed `Interface&` views
- compile-time generics, generic classes/functions, constraints, generic inference subset, and
  guarded monomorphization
- static modules: named imports, aliases, namespace imports, default imports, re-exports, type-only
  imports, project dependencies, and std imports
- C interop subset: checked extern declarations, C header imports for supported declarations, and
  fixed-width ABI aliases
- formatter, CLI diagnostics, LSP diagnostics, hover, symbols, references, formatting, and related
  editor services for the documented subset

## Rejected 0.1 features

These are intentionally unsupported and must fail before C emission:

- JavaScript runtime features: `any`, `unknown`, `never`, implicit `null`, implicit `undefined`,
  truthiness, prototypes, hidden allocation, dynamic property maps, `eval`, async/await, promises,
  exceptions, decorators, JSX, and dynamic imports
- runtime template interpolation
- owning interface values and implicit structural interface objects
- class object headers, vtables, prototype dispatch, heap allocation by `new`, `super`, and
  JavaScript receiver semantics
- sparse arrays, JS array methods, JS iterators, dynamic property access, and runtime module objects
- recursive type aliases, recursive generic instantiations, cyclic imports, and unsupported
  recursive type-shape comparisons
- unsupported C declarator shapes and unsupported runtime layouts, including static-only types in
  stored, parameter, return, or emitted C positions

## C emission contract

TypeC 0.1 emits portable C11 for supported programs. Every translation unit includes fixed-width
primitive aliases via `<stdint.h>`, `<stdbool.h>`, and `<stddef.h>`. TypeC primitive names map
one-to-one to emitted C aliases, for example `i32` emits as `i32` and `usize` emits as `usize`.

See `docs/c-emission.md` for the detailed C emission rules.

## Known limitations

- Generic inference and generic type aliases are intentionally finite subsets.
- Conditional, mapped, intersection, utility-style, and reflection aliases are static checker
  features with documented runtime-layout restrictions.
- Borrowed interface values are explicit non-owning views; owning interface values are not part of
  0.1.
- C header importing accepts only the supported ABI subset.
- LSP features are compiler-backed and intentionally withhold formatting edits for rejected source.

## Release validation

A 0.1 release candidate must pass:

```bash
deno fmt --check
deno task check
deno task lint
deno test -A
deno task build
./bin/STC --version
```

The current 0.1 patch version output must be exactly `TypeC 0.1.2`.
